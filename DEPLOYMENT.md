# KubeCost-Gym Deployment Guide

This guide explains how to deploy KubeCost-Gym to Hugging Face Spaces.

## Prerequisites

- A [Hugging Face](https://huggingface.co) account
- An LLM API endpoint (OpenAI, HuggingFace, or self-hosted)
- API credentials for the LLM service

## Step 1: Create a Hugging Face Space

1. Go to [huggingface.co/spaces](https://huggingface.co/spaces)
2. Click **"Create new Space"**
3. Fill in:
   - **Space name**: `kubecost-gym` (or your preferred name)
   - **License**: Open RAIL-M (recommended)
   - **Select the Space SDK**: **Docker**
   - **Space hardware**: **CPU basic** (2 vCPU, 16GB RAM) ✓ Verified to work
   - Click **"Create Space"**

## Step 2: Clone and Connect to the Space

```bash
# Clone the Space repository (empty initially)
git clone https://huggingface.co/spaces/<your-username>/kubecost-gym
cd kubecost-gym

# Add the KubeCost-Gym repo as a remote
git remote add source https://github.com/your-fork/k8s-cost-optimizer

# Pull all project files
git pull source main

# Push to HF Spaces
git push origin main
```

Alternatively, upload files directly via the HF Spaces UI.

## Step 3: Configure Environment Variables (CRITICAL)

These variables must be set as **Space Secrets** for the deployment to work.

### In HF Spaces Web UI:

1. Go to your Space settings → **"Variables and secrets"**
2. Add these **Secrets** (not regular variables):

#### Required Secret: `HF_TOKEN`

This is the API key for your LLM endpoint (NOT your HuggingFace user token).

**If using HuggingFace's inference router:**
- Go to [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens)
- Create a token with **"Read"** role
- Copy the token value
- Add as `HF_TOKEN` secret in Space settings

**If using OpenAI:**
- Get your API key from [platform.openai.com/account/api-keys](https://platform.openai.com/account/api-keys)
- Add as `HF_TOKEN` secret

**If using another LLM service:**
- Get the API key from your service
- Add as `HF_TOKEN` secret

#### Optional Secrets: `API_BASE_URL` and `MODEL_NAME`

Only set these if using non-default LLM endpoints.

**For HuggingFace's router (default):**
```
API_BASE_URL = https://router.huggingface.co/v1
MODEL_NAME = mistralai/Mistral-7B-Instruct-v0.2:featherless-ai
```

**For OpenAI:**
```
API_BASE_URL = https://api.openai.com/v1
MODEL_NAME = gpt-4-turbo-preview  # or gpt-4, gpt-3.5-turbo
```

**For self-hosted LLM:**
```
API_BASE_URL = https://your-domain.com/v1  # Your API endpoint
MODEL_NAME = your-model-name               # Your model ID
```

## Step 5: Monitor Deployment

1. Go to your Space
2. Watch the **"Build logs"** (shown during build):
   - Should see: `✓ Environment variables validated`
   - Should see: `Launching server on 0.0.0.0:7860`
   - Should see: `OpenEnv REST Endpoints (auto-generated by openenv-core)`
   - Should NOT see any `ERROR` messages

3. Once built, the Space will show status **"Running"**

## Step 6: Use the Application

### REST API Endpoints (OpenEnv-Compliant)

The server provides standard OpenEnv REST endpoints via openenv-core's automatic API generation:

```bash
# Health check
curl https://<your-username>-kubecost-gym.hf.space/health

# Reset environment
curl -X POST https://<your-username>-kubecost-gym.hf.space/reset

# Execute step
curl -X POST https://<your-username>-kubecost-gym.hf.space/step \
  -H "Content-Type: application/json" \
  -d '{"action": "SCALE_REPLICAS(+5)"}'

# Get current state
curl https://<your-username>-kubecost-gym.hf.space/state

# View API docs
https://<your-username>-kubecost-gym.hf.space/docs
```

## Troubleshooting

### 🔴 Space shows "ERROR" or "BUILD_FAILED"

Check the build logs:
1. Click on the Space
2. Click **"Build logs"**
3. Look for error messages

**Common issues:**
- Missing trace files: Ensure `traces/` directory is in repo
- Invalid YAML in `openenv.yaml`: Check syntax
- Missing dependencies: Check `pyproject.toml` is valid

### 🔴 "HF_TOKEN environment variable is required"

**Fix:** Add `HF_TOKEN` as a Secret in Space settings (not a regular variable).

### 🔴 "API_BASE_URL must start with http:// or https://"

**Fix:** Verify your `API_BASE_URL` secret has correct format:
- ✓ `https://api.openai.com/v1`
- ✓ `https://router.huggingface.co/v1`
- ✗ `api.openai.com` (missing https://)

### 🟡 Inference takes a long time or times out

- LLM endpoint may be slow
- Check LLM service status
- Try using a faster/smaller model
- Increase timeout if possible

### 🟡 UI not loading / shows blank page

- Space might still be building (wait 1-2 minutes)
- Check browser console for errors (F12)
- Try hard refresh (Ctrl+Shift+R)

## Advanced Configuration

### Custom LLM Endpoint

If you have a self-hosted LLM endpoint:

1. Set `API_BASE_URL` to your endpoint
2. Set `MODEL_NAME` to your model ID
3. Set `HF_TOKEN` to your API key (if required)

Example for Ollama local server:
```
API_BASE_URL = http://localhost:11434/v1
MODEL_NAME = llama2
HF_TOKEN = (empty or valid token)
```

### Monitoring and Logs

The Space shows live logs in the **"Build logs"** and **"App logs"** sections. Use these to:
- Verify environment variables are loaded
- Debug inference failures
- Monitor performance

## Performance Expectations

On `cpu-basic` hardware (2 vCPU, 8GB RAM):
- **Cold start**: ~30-60 seconds
- **Inference per task**: 2-5 minutes (depends on LLM latency)
- **All three tasks**: 10-15 minutes total
