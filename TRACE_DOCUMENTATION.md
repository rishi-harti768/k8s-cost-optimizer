# Understanding Traces in KubeCost-Gym

## What Are Trace Steps?

**Trace steps** are pre-recorded workload snapshots that define the environment's dynamics at each time point.

### Trace File Structure

```json
{
  "task_name": "cold_start",
  "task_difficulty": "easy",
  "steps": [
    {
      "step": 0,
      "observation": {
        "base_cpu_demand": 45.0,      // Raw CPU demand units
        "base_mem_demand": 60.91,     // Raw memory demand units
        "base_latency_ms": 210.0,     // Baseline latency pressure
        "base_error_rate": 0.6,       // Baseline error pressure
        "base_steal_pct": 0.01,       // Baseline noisy-neighbor pressure
        "active_replicas": 0,         // Expected replica count
        "buffer_depth": 80,           // Request queue depth
        "node_size_class": "S",       // Node tier (S/M/L)
        "current_hourly_cost": 10.0,  // Cost signal
        "node_bin_density": [...]     // Node packing ratios
      },
      "dynamics": {
        "reason": "cold_start_resource_buildout"
      }
    },
    // ... steps 1-49
  ]
}
```

## How Traces Work

### 1. Trace Provides Base Demand
The trace defines the **workload pressure** at each step:
```
Step 0: base_cpu_demand = 45.0
Step 1: base_cpu_demand = 50.0
Step 2: base_cpu_demand = 55.0
...
```

### 2. Agent Takes Action
Your agent decides what to do:
```python
action = agent.decide(observation)
# e.g., SCALE_REPLICAS(+5)
```

### 3. Environment Computes Actual Metrics
The environment combines **trace demand** + **agent capacity** to compute real metrics:

```python
# From env.py _build_observation()
capacity = replicas * node_capacity_multiplier
cpu_usage = (base_cpu_demand / capacity) * 100.0
p99_latency = base_latency_ms * (1.0 + demand_pressure * 0.75)
```

**Example:**
- Trace says: `base_cpu_demand = 100 units`
- Agent has: `5 replicas × 1.0 capacity = 5 units`
- Result: `cpu_usage = (100 / 5) * 100 = 2000%` → **Clamped to 100%**

### 4. Reward Calculated
Based on the actual metrics:
```python
reward = uptime_reward - cost_penalty - sla_breach_penalty + proactive_bonus
```

## Trace Versions

### Primary Traces (v1) - Official Grading

| File | Task | Pattern | Purpose |
|------|------|---------|---------|
| `trace_v1_coldstart.json` | Cold Start | 0→5 replicas | Test reactive scaling |
| `trace_v1_squeeze.json` | Efficient Squeeze | Sinusoidal load | Test sustained efficiency |
| `trace_v1_entropy.json` | Entropy Storm | Noisy-neighbor spikes | Test proactive rebalancing |

**These are used in your inference.py for final scoring.**

### Testing Traces (v2-v5)

#### Cold Start Variants
```
v2_gradual      → Slower ramp-up (easier)
v3_aggressive   → Rapid scaling needed (harder)
v4_failed       → Designed to fail without proper scaling
v5_optimal      → Best-case scenario
```

#### Squeeze Variants
```
v2_steady       → Consistent load (easier)
v3_oscillating  → Fluctuating steal % (harder)
v4_gradual      → Slow increase
v5_optimized    → Efficient resource usage
```

#### Entropy Storm Variants
```
v2_chaos                → Random spikes
v3_cascading            → Sequential failures
v4_reactive_failure     → Reactive approach fails
v5_extreme_chaos        → Maximum difficulty
```

## Why 25 Steps?

Each trace has **exactly 25 steps** (0-24):
- Represents a realistic time window (e.g., 25 minutes, 25 hours)
- Long enough to test sustained behavior
- Short enough for fast evaluation and reduced LLM costs
- Optimized for efficient inference while maintaining task complexity

## Trace Patterns

### Cold Start (v1)
```
Steps 0-5:   High error rate (0.6), low replicas (0-1)
Steps 6-11:  Gradual demand increase (2-3 replicas)
Steps 12-17: Peak demand (4-5 replicas)
Steps 18-24: Sustained load and stabilization
```

### Efficient Squeeze (v1)
```
Sinusoidal pattern over 25 steps:
- cpu_steal oscillates around 20% threshold
- Agent must keep steal < 20% throughout
- Cost pressure increases with replicas
```

### Entropy Storm (v1)
```
Steps 0-7:   Normal operation
Steps 8-13:  cpu_steal rises toward 20%
Steps 14-19: Spike events (steal > 20%)
Steps 20-24: Recovery period

Agent must REBALANCE_NODE *before* steal exceeds 20%
```

## How to Use Different Traces

### In Your Code
```python
# Current (uses v1 traces)
env = KubeCostEnv("traces/trace_v1_coldstart.json")

# Test with harder variant
env = KubeCostEnv("traces/trace_v3_coldstart_aggressive.json")

# Test with easier variant
env = KubeCostEnv("traces/trace_v2_coldstart_gradual.json")
```

### For Local Testing
```python
# Test all cold start variants
for version in [1, 2, 3, 4, 5]:
    trace = f"traces/trace_v{version}_coldstart*.json"
    env = KubeCostEnv(trace)
    # Run your agent
```

### Trace Generation

Traces are generated by `generate_traces.py`:

```python
# Creates 25-step sinusoidal traces
def generate_cold_start_trace():
    steps = []
    for i in range(25):
        demand = compute_sinusoidal_demand(i)
        steps.append({
            "step": i,
            "observation": {
                "base_cpu_demand": demand,
                # ... other metrics
            }
        })
    return {"task_name": "cold_start", "steps": steps}
```

**To regenerate with different step count:**
```bash
set TRACE_STEPS=25
python generate_traces.py
```

## Key Insights

1. **Traces are deterministic** - Same trace always produces same base demand
2. **Agent actions matter** - Your scaling decisions change the actual metrics
3. **Traces test different skills**:
   - v1: Baseline competence
   - v2-v3: Robustness to variations
   - v4: Failure handling
   - v5: Optimal performance

4. **Grading uses v1 only** - But testing with v2-v5 helps you build a robust agent

## Debugging Tips

### View Trace Demand Pattern
```python
import json
import matplotlib.pyplot as plt

with open("traces/trace_v1_coldstart.json") as f:
    trace = json.load(f)

cpu_demand = [s["observation"]["base_cpu_demand"] for s in trace["steps"]]
plt.plot(cpu_demand)
plt.title("CPU Demand Over Time")
plt.show()
```

### Compare Trace Versions
```python
for version in [1, 2, 3, 4, 5]:
    trace_file = f"traces/trace_v{version}_coldstart.json"
    if os.path.exists(trace_file):
        trace = json.load(open(trace_file))
        print(f"v{version}: {len(trace['steps'])} steps")
        print(f"  Max CPU demand: {max(s['observation']['base_cpu_demand'] for s in trace['steps'])}")
```
